# Менеджер финансов компании

Веб-приложение для учёта доходов и расходов компании. Производственная практика.

## Стек
- Python 3.11+, Flask 3, Flask-SQLAlchemy
- SQLite (файл `finance.db` создаётся автоматически)
- Chart.js на фронте
- pytest для тестов

## Возможности
- Учёт счетов (касса, расчётный счёт и т.д.) с авто-обновлением баланса
- Доходы / расходы по категориям, отделам, контрагентам
- Дашборд: сводка за год, помесячная диаграмма, последние операции
- Отчёты с фильтром по датам: P&L, разрезы по категориям и отделам, круговая диаграмма

## Запуск

```bash
python -m venv venv
venv\Scripts\activate          # Windows
pip install -r requirements.txt
python run.py
```

Открыть в браузере: http://127.0.0.1:5000

## Тесты

```bash
pytest
```

## Структура

```
app/
  __init__.py            # фабрика Flask
  database.py            # SQLAlchemy + сидинг справочников
  models.py              # модели БД
  routes/                # blueprint'ы (CRUD + отчёты)
  services/
    transaction_service.py   # бизнес-логика транзакций
    analytics.py             # алгоритмы агрегации
  templates/             # Jinja2-шаблоны (заменяются дизайном из Claude Design)
  static/css/style.css   # базовые стили
tests/                   # pytest
config.py
run.py
```

## Соответствие пунктам производственной практики

| № | Пункт | Где в проекте |
|---|---|---|
| 1 | Алгоритм + средства автопроектирования | ER-диаграмма (drawio), `app/services/analytics.py` |
| 2 | Код по готовой спецификации | `app/models.py`, `app/routes/` |
| 3 | Современный язык + средства отладки | Python 3.11, VS Code debugger |
| 4 | Отладка и тестирование | `tests/` (pytest) |
| 5 | Программный модуль + документация | пакеты `app/services/`, docstrings |
| 6 | Автоматизация документации | Sphinx (`docs/`) |
| 7 | Сбор информации для ВКР | дневник практики |
